4 in 1 Architectural View Model; by Wikipedia; 2023;

"4+1 is a view model used for describing an architecture of software system, based on the use of multiple, concurrent views. The 4+1 view model is rather generic: other notations and tools can be used, other design methods can be used, especially for the logical and process decompositions, but we have indicated the ones we have used with success" - Philippe Kruchten, Architectural Blueprints - The 4+1 View Model of Software Architecture (November, 1995, IEEE Software 12 (6), pp. 42-50); (https://www.cs.ubc.ca/~gregor/teaching/papers/4+1view-architecture.pdf).
