"Ragas Synthetic Test Data Generation"; by Ragas.io; 2024;

Ragas is an open standard for developers to obtain continual learning in their RAG applications. With Ragas, you would be able to:

1.      Synthetically generate a diverse test dataset to evaluate your RAG solution.

2.      Use LLM assisted evaluation metrics to objectively measure RAG performance.

3.      Monitor quality for RAG using smaller models for actionable insights like number of hallucinations in the generated answer.

4.      Use insights to improve your application in your MLOps lifecycle.

Ragas understands that a GenAI evaluation dataset should include the types of questions encountered in production, including questions of varying difficulty levels. LLMs are not good at creating samples with statistical diversity, rather they tend to fall into niche lines of thought.

Questions with different characteristics are required and these are driven by: reasoning, conditioning, context, etc.

Questions need to be created from knowledge found in a provided set of documents (and we add distilled into a knowledge base.  This way test coverage for the entire domain is possible.  Ragas offers metrics tailored to each component of your RAG pipeline in isolation.

These are as follows:

5.      Faithfulness:  How factually accurate are the generated answers;

6.      Answer relevancy: How relevant is the generated answer to the question;

7.      Context recall: Can the solution retrieve all the information required to answer the question;

8.      Context precision: Measures the signal to noise ration of retrieved information;

9.      Context relevancy: Gauges the relevancy of retrieved context, based on the question and information context ; and

10.  Context entity recall: Gives the measure of recall of the retrieved context, based on the number of entities present in both ground truths and contexts relative to the number of entities present in the ground truths.

Refer to https://docs.ragas.io/en/stable/index.html.
