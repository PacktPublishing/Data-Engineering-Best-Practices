Retrieval Augmented Generation (RAG); by Amazon; 2024;

RAG is the  process of augmenting an LLM with domain knowledge.

For example, a chatbot begins with the development of a foundation model (FM), which is an API enabled LLM trained on generalized data.

The user query is converted to a vector query that is run against a domain's knowledge base. The output of the LLM takes the vector query output and into consideration as well as the users context supplied as prompts to generate augmented LLM output. An alternative to RAG exists and that leverages the format of embeddings in the language model itself.

In this approach knowledge base data is converted into a model's acceptable representation and stored in a vector database. This process creates a knowledge library that the generative AI models can understand.

Note that the efficacy of RAG is evident when trying to implement a solution that is of the highest quality as with the goals for semantic search which requires accuracy for knowledge mining tasks (e.g., medical, scientific, legal, and financial searches), where the domain is defined by a vernacular that is vastly different normal discourse (aka based on Latin for doctors) for which the model was trained.
