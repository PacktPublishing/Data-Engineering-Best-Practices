K2view; by K2view; 2024;

K2view has a patented approach to synthetic data generation with the following steps:

1.      First the K2View product models business entities by auto-discovering schemas (e.g., customer, device, loan, order, etc.) for which the synthetic data is needed;

2.      Then K2View generates synthetic data by applying appropriate data generation methods to the data model, in order to create a complete, accurate, and compliant synthetic data set; and

3.      Lastly, K2View delivers the data to a target system and manages access, reservation, versioning, rollback, and integration with DevOps and MLOps pipelines.

K2View supports four key synthetic data generation techniques:

1.      Generative AI creates realistic, synthetic data via ML algorithms.

2.      A rules engine provisions data using user-defined business policies.

3.      Entity cloning extracts entity data, masks it, and then replicates it.

4.      Data masking anonymizes PII / sensitive data to generate compliant data.
