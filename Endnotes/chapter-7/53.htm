Multi-Region AWS Data Lake with Analytics on AWS; October 4, 2022;

1.      Use Amazon Simple Storage Service (Amazon S3) as storage for the data lake;

2.      AWS Glue Data Catalog stores data lake metadata;

3.      Deploy other configurations, such as Amazon Athena workgroup settings or AWS Glue catalog schemas by using infrastructure as code (IaC) pipelines;

4.      Use Multi-AZ configuration for AWS services that store data outside of Amazon S3;

5.      Use Amazon Route 53 or AWS Global Accelerator to provide an endpoint for data ingestion that can switch to another Region during failover;

6.      Use Amazon S3 cross-Region replication to replicate data to the backup Region;

7.      Keep AWS Glue Data Catalog in sync by using IaC pipelines and create new partitions using Amazon S3 events;

8.      Use IaC pipelines to keep configuration in sync;

9.      Use AWS service cross-Region replication to back up other analytics services; and

10.  During failover, the workloads in the Amazon S3 AWS Glue backup region are deployed.
