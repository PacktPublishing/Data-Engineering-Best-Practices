"Three Design Patterns That Use Inversion of Control"; by Alejandro Gervasio, April 14, 2017. Martin Fowler defines InversionOfControl (IoC) – (https://martinfowler.com/bliki/InversionOfControl.html) as implementing the Hollywood Principle (Don't call us, well call you!).

The flow of control between program logic is handed over to a framework system.

The framework decides when to call methods, based on bindings you created when registering components with the framework.

Control is therefore inverted.

The framework calls your code rather than your code calling the framework.
