Pinpoint; 2023;

Pinpoint is used to:

1.      understand application topology and to monitor your application in real time;

2.      gain code-level visibility to transactions;

3.      install APM agents without code; and

4.      have minimal performance impact when running (~3% degradation).

Pinpoint has a few key components like:

1.      ServerMap to understand a solutions topology with reveal data about component status, and transaction count;

2.      Realtime Active Thread Chart to monitor threads in real time for applications;

3.      Request/Response Scatter Chart to visualize requests and response patterns;

4.      CallStack to provide transaction code level visibility for failure analysis;

5.      Inspector to view app details like: app CPU, memory (garbage collection), TPS, and JVM arguments;

6.      URI-Metric hit rates for endpoints; and

7.      Infrastructure system metrics for instances.

Refer to https://github.com/pinpoint-apm/pinpoint.
