"Apache Flink Concept with Examples"; 2024;

Apache Flink needs to be explained with a straightforward introduction to its APIs for managing state and time.

You will need to know:

1.      How to implement streaming data processing pipelines;

2.      How Flink manages state;

3.      How to use event time to compute accurate analytics;

4.      How to build event-driven applications on a continuous stream; and

5.      How to provide fault-tolerant, stateful stream processing with Flink's exactly once semantics (this means that each incoming event affects the final results exactly once even with machine / software failure, there's no duplicate data and no data goes unprocessed).

See: An Overview of End-to-End Exactly-Once Processing in Apache Flink (with Apache Kafka, too!) (https://flink.apache.org/2018/02/28/an-overview-of-end-to-end-exactly-once-processing-in-apache-flink-with-apache-kafka-too) for an explanation of Flink's exactly once semantics in action.

Refer also to https://nightlies.apache.org/flink/flink-docs-master/docs/concepts/overview.
