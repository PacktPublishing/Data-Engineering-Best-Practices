Activity Streams; 2024;

Activity Streams consist of a stream of activities each consisting of an actor, a verb, an object, and a target. See: JSON Activity Streams 1.0 (https://activitystrea.ms/specs/json/1.0).

The stream conveys the activity of a use case as a person performing an action on or with an object.

As an example: Geraldine posted a photo to her album or John shared a video. Components are often explicit, but they may also be implied.

The Activity Streams specification provides sufficient metadata about activities so that a data consumer may present it in a rich human-friendly format. This may include constructing readable sentences about the activity that occurred, visual representations of the activity, or combining similar activities for display.

Within the Activity Streams specification … an object is a thing, real or imaginary, which participates in an activity. It may be the entity performing the activity, or the target entity of the performer.  An object consists of properties.  Some object types refine the meaning of these properties, or they may define additional properties. Some types of objects may have an alternative visual representation in the form of an image, video, or embedded HTML fragments.

A media link represents a hyperlink to such resources.

An Activity Stream is a collection of activities. The relationship between the activities within the collection is undefined by the formal Activity Stream.

Note: activities are serialized using JAVA serializers like the JSON serializer specification in  RFC4627 (https://www.rfc-editor.org/info/rfc4627).

Refer to https://activitystrea.ms.
