Kafka Stream (Confluent); 2024;

Kafka Streams has capabilities not found in open source Kafka (https://kafka.apache.org) like:

1.      Cloud Native Design:

a.      Serverless automated, fully managed Kafka clusters with zero DataOps;

b.      Elastic scaling from 0 to gigabyte per second processing without over-provisioning;

c.      Tiered storage so costs are managed effectively when retaining data at scale without growing compute budget;

d.      High availability at 99.99% SLA uptime with failover & multi-AZ replication;

e.      No Zookeeper metadata management so this open source software need is abstracted away; and

f.        No-touch patching & upgrades optimize infrastructure and support zero-downtime patching / upgrades.

2.      Developer Productivity:

a.      C / C++, Python, Go, .NET clients are supported;

b.      A REST proxy (https://docs.confluent.io/platform/current/kafka-rest) enables access to Kafka from any network application;

c.      A MQTT Proxy (https://docs.confluent.io/platform/current/kafka-mqtt) enables access Kafka from MQTT devices / gateways;

d.      >120 prebuilt stream connectors;

e.      ksqlDB's SQL-based stream processing for on-prem and hybrid workloads; and

f.        A stream designer as visual canvas for building extensible streaming data pipelines with SQL;

3.      Operations:

a.      Kafka management via a cloud UI / control center to manage & monitor clusters at any scale;

b.      Intelligent monitoring & observability via metrics API;

c.      Datadog & Prometheus integration;

d.      Health+ alerting and cloud monitoring;

e.      DevOps Admin APIs;

f.        A Confluent Kubernetes API for deployments;

g.       Ansible playbooks to automate deployments to nay non-containerized environment; and

h.      Self-balancing clusters.

4.      Product vs Stage Requirements:

a.      Granular resource-level authorization of access across user groups;

b.      Security audit logs to detect security threats & anomalies;

c.      Secret protection / encryption for all data-at-rest and network traffic to clients;

d.      Bring your own key to encrypt data-at-rest with your own custom key(s);

e.      Private networking via VPC/VNet peering, AWS Transit Gateway (https://aws.amazon.com/transit-gateway), AWS/Azure Private Link, & GCP Private Service Connect;

f.        HIPAA / GDPR / CCPA readiness through compliance like: SOC 1/2/3 & ISO 27001, GDPR / CCPA readiness, etc;

g.       Data governance via a schema registry as a central registry for data compatibility;

h.      Schema validation for broker-side schema validation & enforcement with topic-level granularity;

i.        Stream catalog for self-service data discovery to search, classify & organize data streams;

j.        Stream lineage to understand data lineage with end-to-end mapping of data streams;

k.       Stream sharing across organizations;

l.        A 99.99% uptime SLA;

m.    Cluster linking to link clusters across any environment to build hybrid & multicloud architectures;

n.      Multi-AZ / multi-region clusters to minimize downtime; and

o.      a replicator used for data replication across clusters.

5.      Support:

a.      24x7x365 expert support for Kafka;

b.      Professional services to build customized outcomes; and

c.      Education / training to build streaming expertise in your organization.

Refer to https://www.confluent.io/what-is-apache-kafka.
