"Apache Beam Capability Matrix"; by Apache Beam Project (https://beam.apache.org); 2024;

 

In the Beam capability matrix you will find mappings to each of these products:

·         Google Cloud Dataflow

·         Apache Flink

·         Apache Spark (RDD/DStream based)

·         Apache Spark Structured Streaming (Dataset based)

·         Apache Samza are mapped to Apache Beam

 

Apache Beam and Google Cloud Dataflow are seen in the table below:

Apache Beam

Google Cloud Dataflow

ParDo

Yes: fully supported

 

Batch mode uses large bundle sizes. Streaming uses smaller sizes.

GroupByKey

Yes: fully supported

Flatten

Yes: fully supported

Combine

Yes: efficient execution

Composite Transforms

Partially: supported via inlining

 

Currently composite transformations are inlined during execution. The structure is later recreated from the names, but other transform level information (if added to the model) will be lost.

Side Inputs

Yes: some size restrictions in streaming

 

Batch mode supports a distributed implementation, but streaming mode may force some size restrictions. Neither mode is able to push lookups directly up into key-based sources.

Source API

Yes: fully supported

 

Support includes Search for "autotuning features" in https://cloud.google.com/dataflow/docs/guides/deploying-a-pipeline.

Metrics

Partially

 

Gauge metrics are not supported. All other metric types are supported.

Stateful Processing

Partially: non-merging windows

 

State is supported for non-merging windows. SetState and MapState are not yet supported.

Refer to https://beam.apache.org/documentation/runners/capability-matrix/what-is-being-computed.
