Google Pub/Sub Dataflow; by Google; 2024;

Google Pub/Sub Dataflow requires some explanation since the Google documentation is very technical, involves open source implementations (aka Apache Beam) and requires a lot of pre knowledge of GCPs software infrastructure.

It requires DataFlow, Pub/Sub, Beam and the IO connector as described below:

1.      A Dataflow (see https://cloud.google.com/dataflow/docs) implementation of the Googles Pub/Sub of the I/O connector:

a.      See: https://cloud.google.com/pubsub/docs/overview; and

b.      See: https://tray.io/documentation/connectors/service/google-cloud-pub-sub.

2.      And Pub/Sub is a scalable, event ingestion / delivery system by Google. Dataflow complements Pub/Sub scalable, at-least-once delivery model with message deduplication, exactly-once processing, and generation of a data watermark (https://stackoverflow.com/questions/42169004/what-is-the-watermark-heuristic-for-pubsubio-running-on-gcd) from timestamped events.

3.      Available as a  Dataflow, when you write a data pipeline using the Apache Beam SDK (https://beam.apache.org) and then execute the pipeline code on the Dataflow service.

Refer to https://cloud.google.com/dataflow/docs/concepts/streaming-with-cloud-pubsub.
